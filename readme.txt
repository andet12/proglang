0) Javascript kan brukes til å gjøre elementer av nettsider dynamiske, forandre på tekst og bilder etter hvor man trykker på siden og generelt manipulere de statiske HTML elementene.

1) ID er unik og forandre kun et element på siden, kan ikke brukes på flere elementer. Class kan brukes på mange elementer på samme side. ID benyttes når elementet skal ha en egen stil, class når flere elementer skal se like ut.

2) Både deloppgave 0 og 1 vil begge bruke mye plass hvis listen av programeringsspråk er lang og begge deloppgavene kan begge være uoversiktlige hvis listen blir lang nok. I deloppgave 1 tar ikke listen opp masse plass på siden når den ikke behøves/er i bruk. Er siden laget kun for å holde oversikt over programmeringsspråk vil det være lettere å finne det språket du leter etter med en model som ligner deloppgave 0.

3) Etter å ha sjekket kildekoden (Ctrl-U) ser det ut til at wikipedia bruker unordered list og mange listed items (<ul> og <il>) skrevet i alfabetisk rekkefølge for å vise proggrammeringsspråkene. De bruker også hver bokstav som overskrift for å holde siden oversiktligere.

4) Div og Span brukes til å adskille elementer så man kan bruke css/javascript på disse elementene alene. Span brukes på små tekstbiter/bilder mens div brukes på større seksjoner av koden.
Div = block-level
Span = Inline

5) De første programmeringsspråkene i listen vil være lette å finne, men skal man lengre ned på listen, spesielt når listen er lengre. Det hjelper at man kan trykke på den bokstaven progspråket man ønsker begynner på for å finne det, men skal man bare se på progspråk generelt vil en løsning hvor man ser alle språkene på en gang nok være mer brukbart.

MVC:

M: Listen over de forskjellige programmeringsspråkene, dataene uten noen handlinger eller grafisk grensesnitt
V: CSS-filen og resten av html-filen, det som bestemmer hvordan ting skal se ut
C: Javascript-filen, det som henter dataene fra modellen å gir det til viewet
